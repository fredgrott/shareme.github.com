Agile Engineering

Developing Mobile Applications demands solving similar and different problems versus the web application development everyone is familiar with. It in fact takes some heavy-intensive knowledge of how the Android Framework works and skills in writing concurrency-thread handling, responsive applications.

Continuous Build Integration

Successful project management depends upon takign huge tasks and breaking them up into very small task that can be managed by observing and measuring the output. My modified Continous Build Integrated system allows me to trakc my code devlopment progress and know at any time the progress of the project as far as the features that work and the featrues still in progress.

Product Discovery

Most think of product discovery as something that happens at the beginning of application development. But, in actuallity product discovery is an on-going process where I talk to users of the application to find out what over-looked features need to be revitialized given theri storng demand from users.

Security Auditing

Securing a Mobile Application to prevent leaking of private data starts before an audit with some basics such as log-wrapping to ensure that no non-needed logging happens in production builds, securing activities and intents, and ensuring that API keys are not leaked via the build system itself or the code within the project.

UI-Customization

UI-widget customization is often required to obtain a specific look and feel for an android mobile application. I like most advance android developer create my own modified UI-widgets to meet specific use cases.

OS-fragmentation

On Android OS fragmentation can be managed and controlled by actively porting features to previous OS versions and maintaing that set of libraries. I actively manage such a library of over 50 UI-components so that I can realtively quarant a specific look and feel of an android application across multiple OS-versions.

During the 1980s and 1990s Cellular Radio Technology was very primitive and not very fast. What began as slow cellular radio standard in Europe named GSM became faster as antennna hardware electronics was shrunk and softtware running the network became more feature compete and faster.

CDMA, code division multiple access, actually first began as concepts in the USSR and was first deployed as a service ofr mobile phones in cars. As other countries also adpoted the technolgy it also ws also depoyd to cars often civil service government units such as police. Qualcomm started improving the technology and came up with some stadnrads that are nicknamed cdmaOne, cdma2000.

GSM, and CDMA

GSM, an European stnadard, its is frequency modulated but ulike FM-radio is modluated by gaussian shift hence the name Gaussian Shift Modulation(GSM).

CDMA, code division multiple access, refers to takign a fast code and xoring the slow data to that fast code to rpduce amodulated signal. Each trasnmitters has a unique fast code and the resulting signals can be multiplexed over one channel which is the multiple access part of CDMA.

Android's Steps Towards Stardom'

Goolge Acquires Android INC

In 2005, Google bought Android Inc with most of the team staying on with Google. During 2005 to 2007 Google decided to come up with a new way of implementing java.

In JavaME java applications are constrained to the applicationlayer and never enter the OS-layer. In pratical implemntation terms that meant that the OEM was never able to upgrade the JavaVM on a javaME powered device much to the disdain by users and security experts. Google came up with the concept of using JNI to bridge between the C++/C parts of the OS and the Java parts and thus could than implement arts of the OS in java.

Google than modified the linux kernel they used to shrink the OS so that they could than plan the OS as being bale to be updated by OEMs. This in part meant that we would see a switch form the mobile device hardware being the selling point to the end user to the the software feature set being the selling pointof that device.

Dalvik Virtual Machine

A new VM

Dalivk VM is named for a small town in Iceland by its creator. Dalivk is implented as a register-based virtual machine.

Thus, how is a register based VM faster than a stack based VM? Okay, some specifics ot java is the fact that java byte-code is 8bit-based. Now for the fun, in Dalvik registers are 16-bits wide which measn that we can have one java code instruction and we have room left over to store the 4-bit address of a local vaiable and thus we get rid of having to do some separate pushes and pops from the operand stack.

The VM Sandbox

Usually, java VMs are sand-boxes but have all the applications execute in that VM and if oen crashes all that applications have to be restarted. With Dalvik and Android OS we have what is known as a Dalvik VM instance that runs one application in its own sand-box. That is why even though you may have 256 Megabytes of device ram you only have the ability to run so many 24meg applications at once.

The OS and Frameworks of Android'

An Android Application

To understand the OS and its frameworks lets go over what an android application represents. An android application can interact with a web browser, it can interact with a database, it can access and interact with sensors of the moible phone, and it can process and interact with media.

Thus, we have certain layers design to support those android application concepts.

Linux Layer

Google modified a stanadard linux kernel to form the foundation of the andorid OS. The modifications were to exclude the X Windows system and include a small subset of GNU standard libraries.

The other part of the linux layer is a set of drivers, some drive hardware and some drive communications.

Anddroid Runtime

The Android Runtime is composed of now three components, the dalvik VM, the JIT,, and the subset of the Apache Harmony Classes.

Framework Layer

The framework layer is composed of frameworks that are a mix of C++ and java that handle such things as SQLite, Webkit, the SurfaceManager, etc, and a Google modified Standard C library called Bionic libc.

Application Layer

The applicaiton sits on top of that stack to provide the api we write applications to to interface with all the featrues and functions of the device while at the saem time managing security and permissions in such a way that we have a use rmanaged sytem of permitting the applicaiton to access device system stuff.

Linux Kernel Layer

The linux kernel layer is a modified linux kernel designed to run on android devices and be cross cpu chip agnostic.

Android Runtime

The android runtime is composed of the Dalvik VM, the JIT, and the Subset of Apache Harmony classes.

Framework Layer

The library layer that implements the features of the Android OS.

Application Layer

The application layer provides the api that we write applications to access.

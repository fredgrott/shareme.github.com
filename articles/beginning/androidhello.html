Android, hello?

Setting up an AVD

For the emulator to work you have to set-up what is called an android virtual device. For right nowwe will not worry about advance settings and just use the defaults.

You will use the Window choice from the IDE main menu and choose AVD manager. When you click on create/new avd it will ask you for an AVD/device size thus just neter 1000M for 1 gig, name your AVD and check on use snapshot and than click finish and your computer will think a second and create the VAD and than show you in a list. You can than highlight that avd and click on start test ot make sure it comes up and runs.

Lets create something

Okay, now on the IDE main menu choose File-New and you will see a long list to your right. Choose android project. In Eclipse 3.7 onward we have to get accustomed to a new way of doing things for DCVS systems(SCM) thus to prevent headaches you will when you get to the create andrid project screen after you name the project and before you click next uncheck save at default location and click on browse.

If you have already setup Mercurial or Git for your OS you will point to the one you will mostly use and clic on create new folder and name the folder for your repo service(ie Guithub, Bitbucket, etc). Now, when you need to create the repo to upload to those services you will create the git repo metada in that folder and the workspace will be linked otinthe correct way. Other-wise you get the metadata folder(.gior.hg) in the wrong location in the folder tree-structure and end up with an extra folder anemd for your project when you upload/push to the remote repo.

Once that is complete click on next and it will give you a coice of api targets, choose android-4 for now, which is android 1.6 and click finish. You will notice that the IDE creates an activity for you with somebasic default template/boilerplate code. Now we are eady to examine the application lifecyle.
